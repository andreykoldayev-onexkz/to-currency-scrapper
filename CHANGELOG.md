# Changelog

## Исправление "Page crashed" в Yandex Cloud

### Проблема
При запуске в контейнере Yandex Cloud Playwright падал с ошибкой:
```
WARNING:scraper:Playwright error: Page.goto: Page crashed
```

### Причина
Chromium требует больше ресурсов (особенно памяти) в контейнерной среде и не может использовать /dev/shm по умолчанию.

### Решение

#### 1. Оптимизация запуска Chromium (app.py)
Добавлены аргументы для работы в контейнере:
- `--disable-dev-shm-usage` - отключает использование shared memory
- `--no-sandbox` - отключает sandbox (безопасно в контейнере)
- `--single-process` - запуск в одном процессе (экономит память)
- `--disable-gpu` - отключает GPU
- Множество других оптимизаций для снижения потребления ресурсов

#### 2. Оптимизация Dockerfile
- Установка только Chromium вместо всех браузеров
- Создание директории для временных файлов
- Удалены секретные данные из ENV (теперь передаются через переменные окружения)

#### 3. Требования к ресурсам
- Минимум 2GB памяти (рекомендуется)
- 1 vCPU
- Timeout 120+ секунд

#### 4. Новые файлы
- `YANDEX_CLOUD_SETUP.md` - детальная настройка для Yandex Cloud
- `DEPLOY.md` - инструкция по деплою
- `docker-compose.yml` - для локального тестирования
- `test_docker.sh` - скрипт тестирования с ограничениями ресурсов
- `.env.example` - шаблон переменных окружения

### Тестирование

Локальное тестирование с ограничениями как в облаке:
```bash
./test_docker.sh
```

Или с Docker Compose:
```bash
docker-compose up --build
```
